# 🎬 视频首尾帧提取工具 (Video Frame Extractor)

一个基于 Python 的图形界面工具，用于快速从视频文件中提取**首帧**和**尾帧**图片，支持单视频和批量模式处理。

---

## ✨ 功能特点

| 功能 | 说明 |
|------|------|
| **🎯 首尾帧提取** | 一键提取视频的首帧和尾帧，保存为高质量 JPG 图片 |
| **⚡ 单视频模式** | 精处理单个视频，实时预览首尾帧效果 |
| **🚀 批量模式** | 支持多视频批量处理，可添加文件夹自动扫描 |
| **📊 视频信息展示** | 自动识别并显示视频的分辨率、帧数、时长等信息 |
| **👀 预览功能** | 提取前实时预览首尾帧缩略图，无需打开外部播放器 |
| **📁 多种格式支持** | MP4、AVI、MOV、MKV、FLV、WMV 等常见视频格式 |
| **🖼️ 图形界面** | 友好的 Tkinter GUI，无需命令行操作 |
| **🔄 进度显示** | 实时显示处理进度和状态 |
| **💪 鲁棒错误处理** | 自动尝试多种策略读取尾帧，避免无效帧 |

## 🚀 快速开始

### 环境要求

- **Python 3.7+**
- Windows / macOS / Linux

### 安装依赖

```bash
pip install opencv-python numpy Pillow
```

或者使用 requirements.txt：

```bash
pip install -r requirements.txt
```

### 运行

```bash
python 首尾帧提取工具.py
```

## 📖 使用指南

### 单视频模式

1. 选择「单视频模式」
2. 点击「浏览视频」选择要处理的视频
3. 点击「选择目录」指定输出位置
4. 查看视频信息和预览图
5. 点击「提取首尾帧」开始处理

### 批量模式

1. 选择「批量模式」
2. 添加视频：
   - 「添加视频」- 多选视频文件
   - 「添加文件夹」- 自动扫描文件夹内的视频
3. 选择输出目录
4. 点击「批量提取」一次性处理所有视频

### 输出文件命名规则

提取的图片会以以下格式保存：

```
视频文件名_首帧.jpg
视频文件名_尾帧.jpg
```

例如：`summer_vlog_首帧.jpg`、`summer_vlog_尾帧.jpg`

## 🧩 项目组成

这个仓库包含**两个独立但互补**的工具：

| 文件 | 功能 | 用途 |
|------|------|------|
| `首尾帧提取工具.py` | 提取视频首尾帧 | 从现有视频获取关键画面 |
| `首尾帧衔接工具.py` | 两张图片生成过渡视频 | 将静态图片合成为动态过渡 |

### 工作流示例

```
视频A ──提取──→ 首帧A + 尾帧A
视频B ──提取──→ 首帧B + 尾帧B
                       ↓
                 [衔接工具]
                       ↓
                尾帧A → 首帧B
                的过渡视频
```

## 🛠️ 技术实现

### 核心依赖

| 库 | 用途 |
|----|------|
| **OpenCV (cv2)** | 视频解码、帧读取、格式转换 |
| **Pillow (PIL)** | 图像保存、格式转换、压缩控制 |
| **Tkinter / ttk** | 跨平台图形界面 |
| **Threading** | 后台处理，防止界面卡顿 |

### 尾帧读取策略

尾帧是一个常见的痛点，很多视频的最后一帧可能损坏或不可读。工具采用了如下策略：

```
1. 尝试读取最后一帧 (total_frames - 1)
2. 如果失败，回退到倒数第二帧
3. 继续回退，最多尝试 3 次
4. 确保总能获取到有效的尾部画面
```

## ❓ 常见问题

**Q: 为什么有些视频的尾帧读取不到？**
A: 一些视频的最后几帧可能损坏，工具会自动向前回退寻找有效帧。如果仍然失败，可能是视频文件损坏。

**Q: 可以自定义输出格式吗？**
A: 当前默认输出 JPEG 格式（质量 95）。如需其他格式（PNG、WEBP）可以修改代码中的 `save()` 调用。

**Q: 支持中文路径吗？**
A: 支持，已处理中文路径编码问题。
